# packs-catalog/

> Deep detail pages for each premium pack. Used on landing site `/packs/*` and as standalone sales assets.

## Naming

`pack-<slug>-detail.md`

Examples:
- `pack-compliance-shield-detail.md`
- `pack-behavioral-intelligence-detail.md`
- `pack-mempalace-pro-detail.md`
- `pack-vertical-os-healthcare-detail.md`
- `pack-vertical-os-legal-detail.md`
- `pack-vertical-os-fintech-detail.md`
- `pack-vertical-os-govtech-detail.md`
- `pack-brand-design-pro-detail.md`
- `pack-agency-command-detail.md`

## Page template (1,200–1,800 words each)

1. **Hero** — pack name + positioning headline + price + primary CTA "Buy License"
2. **The problem** — 2–3 bullet pain points in buyer language
3. **What this pack unlocks** — 5–8 bullets of OUTCOMES (dollar-denominated risk resolved)
4. **What you get** — feature-level deliverables list
5. **Agents unlocked / enhanced** — list with 1-line roles
6. **Skills + commands added** — referencing existing Maxim infrastructure
7. **ROI anchor** — concrete dollar math. Example: "A single GDPR violation averages $250K CAD in fines. This pack is $79/month."
8. **What you keep if you cancel** — trust signal about free-tier fallback
9. **FAQ** — 4–5 pack-specific questions
10. **Buy button** (LemonSqueezy embed)

## Release priority

- `pack-compliance-shield-detail.md` — RELEASE-BLOCKING (v1.0.0)
- All others — POST-Pack-1-revenue (prove one pack works before building six)

## Canonical copy rules

1. Every OUTCOME claim has a dollar amount attached
2. Every capability pulls from free-tier spec v1.2 pack scope (never contradicts)
3. Never use "unlock" without specifying what was locked
4. Never imply the free tier is crippled
5. Always include "What you keep if you cancel" — builds trust

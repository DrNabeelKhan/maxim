# one-pagers/

> Single-page product briefs, feature flyers, sales leave-behinds. Quick hand-out format.

## Naming

`<product-or-feature>-one-pager.md`

Examples:
- `maxim-one-pager.md`
- `voice-mode-one-pager.md`
- `compliance-one-pager.md`

## Artifact types

- Product one-pagers (master, generic)
- Feature one-pagers (deep on one capability)
- Audience one-pagers (same product, different reader)
- Solution one-pagers (use-case focused)

## Template

```markdown
# <Product or Feature> — One-Pager

**Audience:** <enterprise | SMB | developer | investor>
**As-of:** YYYY-MM-DD

---

## The headline (1 sentence)
<The single sentence that captures the value prop.>

## The problem (2–3 sentences)
<Why the reader cares.>

## The solution (3–5 bullets)
- <Capability + outcome>
- <Capability + outcome>
- <Capability + outcome>

## Proof (metrics or quote, ≤2 items)
- <Metric> — <Before> → <After>
- > "<Quote>" — <Name, Role>

## How to get started (single CTA)
<Install link / email / book-a-demo link>

## Contact
<Email / phone / social>

---

## Design notes

- One physical page when printed (US Letter or A4)
- Max ~400 words total
- 1–2 visuals maximum — hierarchy: headline > visual > bullets > CTA
- Export: `.pdf` alongside source markdown
```

## Rules

- ≤ 400 words. Physical one-page constraint.
- Brand Foundation mandatory — voice + banned phrases
- Single CTA only (multiple CTAs = decision fatigue = lower conversion)
- Update revision date on every change; version in filename only if substantially different audience

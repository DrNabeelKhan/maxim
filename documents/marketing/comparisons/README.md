# comparisons/

> Competitive comparison pages — Maxim vs alternatives. Used on landing site `/compare` and as standalone marketing assets.

## Naming

`mxm-vs-<competitor>.md`

Examples:
- `mxm-vs-langgraph.md`
- `mxm-vs-crewai.md`
- `mxm-vs-microsoft-agent-framework.md`
- `mxm-vs-raw-claude.md`
- `mxm-vs-mcp-only.md`
- `decision-flowchart.md`

## Rules

1. **Never bash the competitor.** Factual comparison only. "X does Y; Maxim does Z; here's when each wins."
2. **Acknowledge competitor strengths honestly.** If they're better for a use case, say so. Trust-building > winning every bullet.
3. **Update cadence:** every time a competitor releases a major version. Add a "Last updated" line at top.
4. **Cite sources.** Link to competitor's public docs for every capability claim.
5. **Include decision guidance.** "If you need multi-LLM, use X. If you need Claude-native depth, use Maxim."

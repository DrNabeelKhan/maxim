## Problem

The 88→90 agent count update touched **31 plugin-repo files + 9 landing-page files** in a single sweep. AGENT_SKILL_INVENTORY.md is named "single source of truth" — but in practice the count was hard-coded in:

- 23 markdown files (docs, marketing, guides, ADRs)
- 1 JSON file (`config/agent-registry.json` breakdown comment)
- 9 landing-page TSX/TS files (layout, structured data, opengraph image, pricing, comparisons, etc.)
- 2 proactive-watch teaching examples

[Proactive Watch Class 1 — inventory-drift](../../composable-skills/frameworks/proactive-watch.md) catches **filesystem-vs-inventory** mismatch (e.g., `agents/MXM/cino/` has 4 files but `AGENT_SKILL_INVENTORY.md` says 3). It does NOT catch **inventory-vs-marketing-copy** mismatch (e.g., `AGENT_SKILL_INVENTORY.md` says 90 but `README.md` and `landing-page/app/layout.tsx` say 88).

That gap is how `88` became silent drift waiting to happen.

## Goals

- Future capability count changes touch ONE source of truth.
- Drift between inventory and any consuming surface (docs, marketing, landing page) is detected automatically, not by manual sweep.
- Existing `/mxm-watch` LIGHT scan picks up the gap with no operator effort.
- A `bootstrap/sync-counts` tool propagates changes mechanically when needed (rather than waiting for drift detection to flag them).

## Out of scope

- Replacing static markdown counts with templating (e.g., `{{agent_count}}`) — too invasive, breaks readability of source files for human authors.
- Landing-page build-time injection — separate question, deferred to v1.1.x.
- Per-office sub-counts (CEO 9, CTO 25, etc.) on every surface — only the totals propagate; per-office breakdowns stay in INVENTORY only.

## Three coordinated changes

### Change 1 — Proactive Watch Class 11: `surface-claims-drift`

Insert as Class 11 in [`composable-skills/frameworks/proactive-watch.md`](../../composable-skills/frameworks/proactive-watch.md) § "The 10 Universal Drift Classes" (table grows to 11; rename section heading too).

```
| 11 | **surface-claims-drift** | Marketing/docs/landing-page hard-codes a capability count that doesn't match AGENT_SKILL_INVENTORY.md | COO |
```

**Algorithm (LIGHT phase):**

1. Read `documents/ledgers/AGENT_SKILL_INVENTORY.md` and extract canonical numbers for each tracked counter (Section 1: Agents 90; Section 2: Skill Domains 34; Section 3: Slash Commands 38; Section 4: MCP Servers 7 + Tools 47; Section 5: Hook Scripts 14; Section 6: Frameworks 64; Section 7: Compliance 14).
2. For each counter, scan **declared surface paths** (configurable in `watch-profile.yml`) for hard-coded number literals adjacent to keyword anchors (`agents`, `frameworks`, `commands`, `MCP servers`, `compliance frameworks`, `hooks`, `skill domains`).
3. Pattern: `\b(\d{1,4})[\s ]*(?:\+\s*)?(?:specialist\s+|governed\s+)?{anchor}\b` (case-insensitive; allows `+`-suffix for "90+ agents").
4. Each numeric mismatch with INVENTORY emits a drift event of severity 3 (warn-only in LIGHT; FULL phase auto-routes to COO review queue).

**Surface scan paths (default):**

```yaml
surface-claims-drift:
  enabled: true
  source_of_truth: documents/ledgers/AGENT_SKILL_INVENTORY.md
  surface_paths:
    - "*.md"
    - "documents/**/*.md"
    - ".claude/**/*.md"
    - "templates/**/*.md"
    - "distributions/**/*.md"
    - "packaging/**/*.md"
    - "composable-skills/**/*.md"
    - "config/agent-registry.json"
  excluded_paths:
    - "CHANGELOG.md"          # historical version entries are point-in-time accurate
    - "documents/ADRs/INDEX.md"  # historical superseded ADRs
    - "**/v[0-9]*-*.md"       # versioned historical docs (e.g., maxim-pack-catalog-v1.0.0.md)
    - "**/changelog/**"        # any changelog directory
    - "**/migration-log*"      # operational ledgers preserve historical state
  anchors:
    - keyword: "agents"
      sources: ["Section 1 — Specialist Agents"]
    - keyword: "frameworks"
      sources: ["Section 6 — Behavioral Frameworks"]
    - keyword: "compliance frameworks"
      sources: ["Section 7 — Compliance Frameworks"]
    - keyword: "skill domains"
      sources: ["Section 2 — Domain Skills"]
    - keyword: "commands"
      sources: ["Section 3 — Slash Commands"]
    - keyword: "MCP servers"
      sources: ["Section 4 — MCP Servers"]
    - keyword: "MCP tools"
      sources: ["Section 4 — MCP Servers"]
    - keyword: "hooks"
      sources: ["Section 5 — Hooks"]
```

**Landing-page scan paths (separate config):**

The `landing-page/` repo is a sibling project that consumes Maxim's identity but is not under plugin-repo. The proactive-watch checker resolves landing-page via `cross_repo_targets` in `watch-profile.yml`:

```yaml
surface-claims-drift:
  cross_repo_targets:
    - path: "../landing-page"
      file_globs:
        - "**/*.tsx"
        - "**/*.ts"
        - "lib/**/*"
        - "components/**/*"
      excluded:
        - "app/changelog/page.tsx"  # historical release entries
        - "app/roadmap/page.tsx"    # historical timeline entries
        - "app/giveaway/**"          # currency values $X.88 false-match risk
        - "components/DispatchDiagram.tsx"  # SVG coordinates
```

**Anchor false-positive guard:** scan must distinguish currency (`$X.88`), SVG coordinates (`y: 88`), and percentages (`88%`) from agent-count claims. Pattern explicitly requires the keyword anchor adjacent (`88 agents`, not bare `88`). Coordinates and currency don't have the keyword adjacent.

### Change 3 — `bootstrap/sync-counts.{sh,ps1}` tool

New executable hook + bootstrap utility. Sketch:

```sh
#!/usr/bin/env bash
# bootstrap/sync-counts.sh — propagate AGENT_SKILL_INVENTORY counts across all surfaces.
# Idempotent: running twice is a no-op.
# Exit codes: 0 = clean (or successfully synced), 1 = unrecoverable drift, 2 = invalid inventory

set -euo pipefail

REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
INVENTORY="$REPO_ROOT/documents/ledgers/AGENT_SKILL_INVENTORY.md"
LANDING_PAGE_ROOT="${MAXIM_LANDING_PAGE:-$REPO_ROOT/../landing-page}"

# 1. Parse INVENTORY
AGENTS=$(grep -E '^## Section 1' "$INVENTORY" -A 20 | grep -oP 'Total agents:\s*\K\d+' | head -1)
SKILLS=$(grep -E '^## Section 2' "$INVENTORY" -A 10 | grep -oP 'Total domain skills:\s*\K\d+' | head -1)
COMMANDS=$(grep -E '^## Section 3' "$INVENTORY" -A 10 | grep -oP 'Total commands:\s*\K\d+' | head -1)
# ...etc for MCP servers, frameworks, compliance, hooks

[[ -z "$AGENTS" ]] && { echo "FATAL: could not parse Section 1 agent count from $INVENTORY"; exit 2; }

# 2. For each (count, anchor) pair, sed-substitute across declared surfaces
sync_count() {
  local count=$1 anchor=$2 surfaces=("${@:3}")
  for surface in "${surfaces[@]}"; do
    [[ ! -f "$surface" ]] && continue
    # Replace (number)+optional+(space)+(anchor) with new count, only when adjacent
    perl -i -pe "s/\b\d{1,4}(\+?)(\s+(?:specialist\s+|governed\s+)?$anchor)\b/${count}\$1\$2/gi" "$surface"
  done
}

# Read excluded_paths + scan_paths from watch-profile.yml → bash array (yq or python helper)
# Run sync_count for each anchor

# 3. Verify post-sync state — re-run proactive-watch class 11 in LIGHT phase
# If any drift remains, exit 1 with details (surface tool can't auto-update flagged for manual review)

echo "✓ Counts synchronized: $AGENTS agents, $SKILLS skills, $COMMANDS commands, ..."
```

PowerShell sibling at `bootstrap/sync-counts.ps1` mirrors the same logic for Windows.

**Idempotency guarantee:** running `sync-counts` when nothing has changed is a no-op (no file modifications, exit 0). Running after a count bump produces exactly the diff that the prior manual sweep produced.

## Risks

- **Cross-repo scan (`landing-page/`):** sync-counts reaching into a sibling repo to edit TSX files is unusual. Mitigation: `MAXIM_LANDING_PAGE` env var must point to the landing-page checkout; if not set, landing-page is silently skipped (not flagged as drift). Operator explicitly opts in by setting the env.
- **Anchor regex false matches:** number-keyword adjacency catches `88 agents` correctly but might miss `eighty-eight agents` (spelled out). Mitigation: scope the v1.0.x version to numeric-only; spelled-out numbers are rare in current corpus and can be flagged manually.
- **Markdown table cells with multiple counts on one line:** e.g., the `documents/marketing/catalogues/` capability matrix has `90 Maxim agents` in a table cell. Pattern handles this; tested against current corpus.
- **Versioned historical docs:** `maxim-pack-catalog-v1.0.0.md` is named with v1.0.0 semantics frozen at that release. The exclusion pattern `**/v[0-9]*-*.md` skips these. Same for `maxim-catalogue-v1.0.0.md`.

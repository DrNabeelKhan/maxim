## Full Product Development Cycle

Run these in order for any feature, product, or project.

| Step | Command | What it does |
|---|---|---|
| 1 | `/mxm-plan` | Breaks your task into a clear step-by-step plan. Always start here for anything bigger than a quick fix. |
| 2 | `/mxm-implement` | Builds what the plan describes. Routes to the right CTO specialists automatically. |
| 3 | `/mxm-review` | Checks the work before it goes further. Flags issues, applies standards, loops in security if needed. |
| 4 | `/mxm-test` | Runs quality checks — unit tests, integration tests, load tests. Pass means ready to ship. |
| 5 | `/mxm-release` | Prepares the release — changelog, final checklist, security sign-off. Nothing ships without this step. |

**Full cycle example:**
```
/mxm-plan build a user authentication system with OAuth
→ /mxm-implement
→ /mxm-review
→ /mxm-test
→ /mxm-release
```

## Specialist Commands

Activate any time — independent of the office shortcuts.

### `/mxm-new-project`
Interactive project scaffold — creates a new project folder with all Maxim per-project files, placeholder templates, and `.claude-sessions-memory\` session storage. Works from Claude Code Desktop or CLI without PowerShell.
```
/mxm-new-project
```
Maxim asks: folder name → parent location → display name → vertical → compliance markets → stage. Confirms before creating anything. Creates 11 files + 4 folders.

### `/mxm-design`
Activates the full design team at once — UX, UI, brand, design system, and behavioral science all working together.
```
/mxm-design create the onboarding flow for new users
```

### `/mxm-seo`
SEO, AEO, and GEO — search visibility across Google, voice search, and AI-generated answers.
```
/mxm-seo audit our website and improve our search rankings
```

### `/mxm-security`
Security audit, threat modeling, penetration testing, and compliance check.
```
/mxm-security review our API gateway for vulnerabilities
```

### `/mxm-compliance`
GDPR, HIPAA, SOC2, ISO 27001 — regulatory review against your declared compliance frameworks.
```
/mxm-compliance check if our data handling meets GDPR requirements
```

### `/mxm-behavior`
Applies behavioral science — Fogg model, nudge theory, habit loops — to reduce friction and improve user adoption.
```
/mxm-behavior reduce drop-off in our sign-up flow
```

### `/mxm-superpowers`
Advanced workflows — parallel specialists, systematic debugging, test-driven development, and verification.
```
/mxm-superpowers debug this regression and find the root cause
```

### `/mxm-remember`
Store important context in MemPalace with automatic office routing. Dual-writes to both MemPalace and `.claude-sessions-memory/` files.
```
/mxm-remember we decided to use PostgreSQL for the audit log instead of MongoDB
```

### `/mxm-recall`
Retrieve context from MemPalace by topic, project, or office. Falls back to file-based search when MemPalace is unavailable.
```
/mxm-recall what did we decide about the auth architecture?
```

### `/mxm-status`
Project-level status — shows active agents, skill gaps, review queue, current task state, and compliance status.
```
/mxm-status
```

### `/mxm-health`
Full 8-layer system diagnostic — tests MCP servers, memory layers, Claude Desktop config, portfolio, scheduled tasks, and **offers to autofix** any issues found.
```
/mxm-health
```

### `/mxm-update`
Update the current project's Maxim configuration to the latest version. Reads all existing data, shows a gap report, and applies changes with confirmation.
```
/mxm-update
```
Detects version gap, scans planning/memory files, merges manifest, creates missing files. Never overwrites existing data.

| Content writing, articles, SEO copy | `/mxm-cmo` | Content Strategist · Documentation Writer |

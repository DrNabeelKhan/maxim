# Brand Design Library — Awesome DESIGN.md Collection

> 59 brand DESIGN.md files extracted from real production websites. Each follows the Google Stitch DESIGN.md 9-section format: visual theme, color palette, typography, component stylings, layout, depth/elevation, do's/don'ts, responsive behavior, and agent prompt guide.

**Source:** `community-packs/design-templates/` (VoltAgent/awesome-design-md, MIT license)
**Online catalog:** https://getdesign.md
**Usage:** Copy the brand's DESIGN.md to your project root and tell Claude: "build me a page that looks like this."

---

## How to Use a Brand Reference

1. Find the brand below that matches the desired aesthetic
2. Read the full DESIGN.md at: `community-packs/design-templates/design-md/[brand]/README.md`
   - Note: Local README.md files link to the full hosted version at `https://getdesign.md/[brand]/design-md`
3. Copy the full DESIGN.md content into your project's `DESIGN.md`
4. Instruct the AI agent to build UI following that design system

## DESIGN.md Section Structure

Every brand DESIGN.md follows the Google Stitch 9-section format:

| # | Section | What it captures |
|---|---------|-----------------|
| 1 | Visual Theme & Atmosphere | Mood, density, design philosophy |
| 2 | Color Palette & Roles | Semantic name + hex + functional role |
| 3 | Typography Rules | Font families, full hierarchy table |
| 4 | Component Stylings | Buttons, cards, inputs, navigation with states |
| 5 | Layout Principles | Spacing scale, grid, whitespace philosophy |
| 6 | Depth & Elevation | Shadow system, surface hierarchy |
| 7 | Do's and Don'ts | Design guardrails and anti-patterns |
| 8 | Responsive Behavior | Breakpoints, touch targets, collapsing strategy |
| 9 | Agent Prompt Guide | Quick color reference, ready-to-use prompts |

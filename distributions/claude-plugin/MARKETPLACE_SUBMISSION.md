## Review quality signals (for Anthropic reviewers)

Maxim differentiates from generic agent libraries on five axes. This section surfaces the quality signals a reviewer would check.

### 1. Framework citation and enforcement

Every agent, skill, and pack SKILL.md cites a peer-reviewed behavioral framework with the mechanism stated. See `documents/reference/FRAMEWORKS_MASTER.md` (64 frameworks) and the 6 L1 pack SKILL.md files in `mxm-packs-source/packs/pack-l1-{1..6}-*/SKILL.md` (one distinct primary framework per pack: Prospect Theory, Cognitive Load Theory, Signal Detection Theory, COM-B, Dual Process Theory, Fogg B=MAP).

### 2. Governance as enforcement, not documentation

The CSO auto-loop fires automatically on regex-matched security/compliance/PII/regulated prompts. The license hook emits permissionDecision JSON per Claude Code plugin spec. The behavioral-moat prompt hook refuses to ship external-facing output that does not apply at least one named framework with mechanism stated.

### 3. Audit trail as the defensibility

Every AI decision generates a structured activity record. The `activity.jsonl` layer is the evidence that makes governance auditable rather than aspirational. See `documents/ledgers/MOAT_TRACKER.md` MOAT-01 (AI Governance) for the mechanism claim.

### 4. ADR coverage

10 Architecture Decision Records ratified. All ADRs present in `documents/ADRs/` (INDEX + TEMPLATE + individual records). ADR-007 documents the behavioral-moat framing doctrine that governs all SKILL.md authoring.

### 5. Proactive drift detection

4 free drift-detection monitors ship with the plugin (inventory, version, cross-doc, stale-handoff). 8 additional drift classes are gated to L1.3 Proactive Watch pack. Signal Detection Theory per-class operating points documented in the L1.3 pack SKILL.md.

## Known limitations at v1.0.0 submission

1. **Windows shell dependency** — monitors use `bash -c` loops. Windows users need Git Bash. Resolution: Sprint 8b (v1.0.0) migrates monitors to `mxm-pack-engine watch <class>` Go subcommand, eliminating the Git Bash dependency.
2. **Platform coverage** — host-platform binary build only for initial submission. `-Release` flag produces 4 cross-platform binaries (linux/amd64, windows/amd64, darwin/arm64, darwin/amd64). Invoke `build-claude-plugin.ps1 -Release` before submission.
3. **License endpoint DNS** — `license.isystematic.com` DNS points to Cloudflare Worker. Two Cloudflare Worker secrets remain pending LemonSqueezy store approval (documented in `mxm-packs-source/cloudflare-worker/DEPLOY_CHECKLIST.md`).
4. **Output style** — `mxm-mode.md` is the only output style; additional styles (`compliance-mode`, `executive-summary`) planned for v6.5.0.

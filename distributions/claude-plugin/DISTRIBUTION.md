# Maxim — Claude Code Plugin Distribution

**Version:** 6.4.4
**License:** BSL 1.1 (4-year Apache 2.0 conversion per ADR-005)
**Namespace:** `maxim`

Maxim packaged as a Claude Code plugin. Installs the full behavioral intelligence layer: 90+ agents across 7 executive offices, 64 peer-reviewed frameworks, governance hooks, proactive drift detection, and the three-voice .brand-foundation load.

## Installation

```
/plugin install maxim@anthropic-official
```

After installation, `/plugin list` confirms the plugin is active. The plugin loads the Maxim dispatch sequence, adds 37+ slash commands, and activates the `executive-router` agent for every new conversation.

## Prerequisites

- **Claude Code** 2.1 or later (plugin support required)
- **Windows users:** Maxim background monitors require a POSIX shell. Install Git for Windows (https://git-scm.com/download/win) — ships with Git Bash. Mac and Linux have bash natively.
- **Go** 1.22 or later (only if building from source; end users install pre-compiled binaries)
- **Node.js** 18 or later (for MCP servers)

### License activation flow

```bash
# 1. Purchase pack on maxim.isystematic.com (powered by LemonSqueezy)
# 2. Receive JWT license via email
# 3. Activate locally:
mxm-pack-engine activate --license <JWT>

# 4. Verify:
mxm-pack-engine fingerprint
mxm-pack-engine refresh    # re-validates against license.isystematic.com
```

The JWT is cached at `~/.mxm-packs/license.jwt`. The plugin's license hook reads this file on every tool permission request and validates against the endpoint. No license = free tier only (never deadlocks).

## Distributor build process

Plugin maintainers can rebuild the distribution from canonical source:

```powershell
# From maxim repo root:
.\bootstrap\build-claude-plugin.ps1           # host-platform build
.\bootstrap\build-claude-plugin.ps1 -Clean    # clean mirror first
.\bootstrap\build-claude-plugin.ps1 -Release  # 4 cross-platform binaries
.\bootstrap\build-claude-plugin.ps1 -DryRun   # preview operations
```

The build script:
1. Mirrors canonical `.claude/skills/`, `.claude/commands/`, `agents/`, `mcp/`, and `.claude/hooks/` scripts into `distributions/claude-plugin/`
2. Compiles `pack-engine` Go binary from `pack-engine/` source
3. Generates plugin-internal `.mcp.json` with paths rewritten to `${CLAUDE_PLUGIN_ROOT}/mcp/<server>/server.js`

Generated artifacts are gitignored until a release branch is cut.

## Update

```
/plugin update maxim
```

Updates follow semantic versioning. Breaking changes ship in major versions only (v7.0+).

## Uninstall

```
/plugin uninstall maxim
```

Removes the plugin, its skills, agents, commands, hooks, monitors, and output style. Does not remove `~/.mxm-packs/license.jwt` (preserved for re-install).

# Maxim Bootstrap Templates

Use these templates to initialize any new project with the correct Maxim skill stack, external references, workflow patterns, and planning structure pre-seeded.

Maxim agents repo is always the starting point for every new project. Copy the relevant bootstrap folder into your new project repo and follow the CHECKLIST.md inside.

## Available Templates

| Template | Maps To | Use When |
|---|---|---|
| `saas-platform/` | FixIt, DrivingTutors.ca | Building a marketplace, subscription SaaS, or multi-sided platform |
| `ai-agent-service/` | iSimplification | Building an AI agent platform, LLM service, or agentic workflow product |
| `compliance-app/` | SentinelFlow, GulfLaw.ai | Building a regulated-industry app (healthcare, legal, finance, government) |
| `api-backend/` | Generic | Building a standalone API, microservice, or backend service |

## How To Use

1. Copy the bootstrap folder for your project type into your new repo root
2. Rename `CHECKLIST.md` → follow every item in order
3. Replace all `{{PLACEHOLDER}}` values with your project specifics
4. Delete this README from your project repo (it’s for Maxim reference only)

## Maxim Supremacy Reminder

Every bootstrap template pre-loads the Maxim dispatch. The Maxim skill stack is always Layer 1. External sources (alirezarezvani, superpowers, VoltAgent) are always subordinate. Do not remove or reorder the skill activation sequence in any template.
